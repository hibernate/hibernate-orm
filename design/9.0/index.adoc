= ORM 9 Boot Model Design Notes
:toc:

This directory collects design notes for the ORM 9 boot-model work.

The documents are not all the same kind of artifact.  Some describe durable
target design, some are contributor references, some are completed audits, and
some are historical notes retained only for rationale.  Do not use every file in
this directory as current branch state.

== Start Here

link:current-work-handoff.adoc[`current-work-handoff.adoc`]::
The only current-status document.  Read this first when picking up an existing
worktree, starting a new chat, or asking "what should we do next?"

link:design-doc-audit.adoc[`design-doc-audit.adoc`]::
The cleanup and routing map for this directory.  It classifies every design
note as active, reference, resolved, historical, deferred, or an archive
candidate, and names the canonical document for each design area.

Use the audit before updating older design notes.  If a document and the audit
disagree about whether a topic is active, resolved, or historical, treat the
audit as the routing source and fix the stale document before adding more
status.

== Current Policy

* Current branch status, test health, and next-work guidance belong in
  link:current-work-handoff.adoc[`current-work-handoff.adoc`].
* Durable target design and rationale belong in focused design notes.
* Contributor orientation belongs in reference docs.
* Historical PoC notes and work logs should be folded into canonical docs or
  removed once their useful decisions are captured.
* Do not add new status narratives to design docs other than the handoff.
