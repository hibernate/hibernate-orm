= Still Missing From Model Building
:toc:

This document tracks known remaining gaps in the ORM 9 model-building prototype:
source discovery, annotation/XML categorization, phased binding, and production
of `org.hibernate.mapping` metadata.

The intent is to keep model-building implementation gaps separate from broader
ORM integration notes.  Some items here may eventually imply mapping-model API
changes, but they first need a clear local prototype story.

NOTE: This gap analysis is based on an older version of ORM code, probably even pre-8.0.  The analysis will need to be performed again.

See also `annotation-coverage.adoc` for the broader audit of runtime annotations
declared in `jakarta.persistence` and `org.hibernate.annotations`, including
`DialectOverride`.

== Removed Annotation Vestiges

`@SelectBeforeUpdate` was removed as a Hibernate annotation and should not be
treated as an annotation coverage item for the new pipeline.

Remaining vestiges are legacy/XML/runtime mapping contracts:

* `org.hibernate.mapping.PersistentClass#setSelectBeforeUpdate(boolean)` and
  `hasSelectBeforeUpdate()`
* `org.hibernate.boot.jaxb.mapping.spi.JaxbEntity#isSelectBeforeUpdate()`
* `org.hibernate.boot.jaxb.hbm.spi.EntityInfo#isSelectBeforeUpdate()`
* `org.hibernate.boot.model.source.spi.EntitySource#isSelectBeforeUpdate()`
* persister/runtime reads of the resulting mapping flag

Parity/removal plan:

1. Audit whether JPA `orm.xml` should retain any `select-before-update`
   vocabulary.  If not, remove it from mapping XSD/JAXB processing rather than
   carrying it into the new source model.
2. Treat HBM `select-before-update` as legacy-only.  If HBM support is removed
   from the ORM 9 boot-model path, remove the HBM source contract vestige as
   part of that cleanup.
3. After XML/HBM source decisions are settled, remove or deprecate the
   `PersistentClass` flag and persister plumbing if no supported source can
   set it.
4. Until then, do not add new categorization/binding support for a removed
   annotation; any remaining work belongs to XML/source compatibility or
   mapping-model cleanup.
