= Still Missing From Model Building
:toc:

This document tracks known remaining gaps in the ORM 9 model-building prototype:
source discovery, annotation/XML categorization, phased binding, and production
of `org.hibernate.mapping` metadata.

The intent is to keep model-building implementation gaps separate from broader
ORM integration notes.  Some items here may eventually imply mapping-model API
changes, but they first need a clear local prototype story.

NOTE: This gap analysis is based on an older version of ORM code, probably even pre-8.0.  The analysis will need to be performed again.

== Association Identifiers and Derived Identifiers

Known remaining gaps:

* Inverse `@OneToOne` association identifiers.
+
`IdClass` association identifiers currently support owning to-one attributes.
The inverse `@OneToOne(mappedBy = ...)` case still fails explicitly.  This likely
belongs near the inverse to-one association phase, because the identifier value
needs to be reconciled with the owning-side property after members and table keys
exist.
+
Prototype touchpoints:
+
* link:../../hibernate-core/src/main/java/org/hibernate/boot/models/bind/internal/binders/IdentifierBinder.java[`IdentifierBinder`]
* link:../../hibernate-core/src/main/java/org/hibernate/boot/models/bind/internal/binders/InverseToOneAssociationBinder.java[`InverseToOneAssociationBinder`]
* link:../../hibernate-core/src/test/java/org/hibernate/orm/test/boot/models/bind/id/SimpleIdTests.java[`SimpleIdTests`]

== Annotation and XML Coverage

These are annotation and XML coverage items that have not yet been promoted into
a focused implementation note, an explicit binder gap above, or focused tests.

Parity-required annotation support:

* `@OnDelete`
* `@Parent`
* `@PartitionKey`
* `@RowId`
* `@TimeZoneStorage`
* `@TimeZoneColumn`
* `@CreationTimestamp`
* `@UpdateTimestamp`
* generated values

ORM 9 nice-to-have annotation support:

* `@Struct`
* `@View`
* hierarchical package metadata
+
Open question: should mappings defined on package `com.acme` apply to
`com.acme.model`?  If so, should `PackageDetails` expose its parent package?

XML parity and XSD coverage:

* `@LazyGroup` in XSD
* `@ListIndexBase` in XSD
* `@Mutability` in XSD
* `@SQLInsert`, `@SQLUpdate`, and `@SQLDelete` for secondary tables in XSD

XML/dynamic-model-only vocabulary:

* `<mapped-superclass/>` support in dynamic models
* `@Abstract`
* `@Extends`
* `@CollectionClassification`
* `@Target`
+
`org.hibernate.annotations.Target` exists, but is deprecated.  We need to decide
whether to repurpose it for XML/dynamic-model use or define a replacement.

Resolved or covered locally:

* `@SecondaryRow`
* `@SoftDelete`
* `@TenantId`
